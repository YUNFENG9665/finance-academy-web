金融模型回测框架

回测的本质

回测是用历史数据模拟策略的实际执行，评估策略的有效性。但回测不是预测未来，而是检验策略在历史上是否有效。

回测的核心原则
原则含义违反后果检查方法
无未来信息决策时只能用当时可用数据严重高估收益检查数据时间戳
真实成本包含交易成本、滑点、冲击虚假盈利实际交易数据验证
样本外测试用未参与训练的数据测试过拟合严格时间分割
多周期验证在不同市场环境下测试策略脆弱分周期分析
参数敏感性检验结果对参数的依赖参数过拟合参数扫描分析

过拟合与稳健性检验

过拟合的识别

过拟合是回测中最大的敌人——策略在历史数据上表现完美，但在新数据上失效。

过拟合的信号
训练-测试差距大
表现: 样本内夏普>2，样本外<0.5
检验方法: 严格样本外测试
参数敏感
表现: 参数微调导致结果剧变
检验方法: 参数敏感性分析
逻辑不清
表现: 无法解释策略为何有效
检验方法: 经济逻辑审查
交易过频
表现: 依赖噪音而非信号
检验方法: 换手率分析
样本量不足
表现: 少量交易撑起高收益
检验方法: 统计显著性检验

def parameter_sensitivity(strategy, param_name, param_range):
    """检验策略对参数变化的敏感程度"""
    results = []
    for param_value in param_range:
        sharpe = strategy.backtest(param_name=param_value)
        results.append(sharpe)

    # 好的策略：结果应该平滑变化，而非剧烈跳跃
    return np.std(results)  # 标准差越小越稳健

def rolling_backtest(strategy, window_size=252, step=63):
    """在不同时间窗口检验策略有效性"""
    sharpes = []
    for start in range(0, len(data) - window_size, step):
        window_data = data[start:start+window_size]
        sharpe = strategy.backtest(window_data)
        sharpes.append(sharpe)

    # 好的策略：不同窗口表现一致
    return np.mean(sharpes), np.std(sharpes)

def bootstrap_test(returns, n_samples=1000):
    """估计策略收益的置信区间"""
    sharpes = []
    for _ in range(n_samples):
        sample = np.random.choice(returns, size=len(returns), replace=True)
        sharpe = np.mean(sample) / np.std(sample) * np.sqrt(252)
        sharpes.append(sharpe)

    return np.percentile(sharpes, [2.5, 97.5])  # 95%置信区间

样本量计算
\[n = \frac{2(z_{\alpha/2} + z_\beta)^2 \sigma^2}{\delta^2}\]

其中：

•\(z_{\alpha/2}\)：显著性水平对应的z值（通常1.96）
•\(z_\beta\)：统计功效对应的z值（通常0.84对应80%功效）
•\(\sigma\)：指标的标准差
•\(\delta\)：最小检测效应（MDE）

PSI计算
\[PSI = \sum_{i=1}^{n} (Actual_i - Expected_i) \times \ln\left(\frac{Actual_i}{Expected_i}\right)\]
python
def calculate_psi(expected, actual, bins=10):
    """计算PSI（Population Stability Index）"""
    # 分箱
    breakpoints = np.percentile(expected, np.linspace(0, 100, bins+1))

    # 计算各箱占比
    expected_percents = np.histogram(expected, breakpoints)[0] / len(expected)
    actual_percents = np.histogram(actual, breakpoints)[0] / len(actual)

    # 避免除零
    expected_percents = np.clip(expected_percents, 0.001, None)
    actual_percents = np.clip(actual_percents, 0.001, None)

    # PSI
    psi = np.sum((actual_percents - expected_percents) *
                 np.log(actual_percents / expected_percents))

    return psi

关键模型与公式

1

模型1：回测收益计算

简单收益率
\[r_t = \frac{P_t - P_{t-1}}{P_{t-1}}\]

对数收益率（推荐，可加性）：

\[r_t = \ln\left(\frac{P_t}{P_{t-1}}\right)\]
考虑成本的净收益
\[r_{net} = r_{gross} - tc \times |w_t - w_{t-1}|\]

其中\(tc\)是交易成本率，\(w\)是权重变化。

2

模型2：夏普比率的统计检验

夏普比率的标准误
\[SE(SR) = \sqrt{\frac{1 + \frac{SR^2}{2}}{n}}\]
t检验
\[t = \frac{SR}{SE(SR)} = SR \times \sqrt{\frac{n}{1 + \frac{SR^2}{2}}}\]

经验法则：年化夏普比率1.5以上需要约2-3年数据才能达到统计显著。

3

模型3：最大回撤计算

python
def max_drawdown(returns):
    """计算最大回撤及持续时间"""
    cum_returns = (1 + returns).cumprod()
    running_max = cum_returns.cummax()
    drawdown = (cum_returns - running_max) / running_max

    max_dd = drawdown.min()

    # 回撤开始和结束时间
    end_idx = drawdown.idxmin()
    start_idx = cum_returns[:end_idx].idxmax()

    return max_dd, start_idx, end_idx

class RobustBacktester:
    def __init__(self, data, train_ratio=0.6, val_ratio=0.2):
        # 严格的时间分割
        n = len(data)
        self.train_data = data[:int(n*train_ratio)]
        self.val_data = data[int(n*train_ratio):int(n*(train_ratio+val_ratio))]
        self.test_data = data[int(n*(train_ratio+val_ratio)):]

    def run_backtest(self, strategy, transaction_cost=0.003, slippage=0.001):
        """
        包含真实成本的回测
        """
        results = []

        for data, name in [(self.train_data, 'train'),
                           (self.val_data, 'val'),
                           (self.test_data, 'test')]:
            # 生成信号
            signals = strategy.generate_signals(data)

            # 计算收益（含成本）
            returns = self._calculate_returns_with_costs(
                data, signals, transaction_cost, slippage
            )

            # 计算指标
            metrics = self._calculate_metrics(returns)
            metrics['dataset'] = name
            results.append(metrics)

        return pd.DataFrame(results)

    def _calculate_returns_with_costs(self, data, signals, tc, slippage):
        """考虑交易成本和滑点的收益计算"""
        position_changes = signals.diff().abs()

        # 交易成本
        costs = position_changes * tc

        # 滑点（与交易量成正比的额外成本）
        slippage_costs = position_changes * slippage

        # 净收益
        gross_returns = data['returns'] * signals.shift(1)
        net_returns = gross_returns - costs - slippage_costs

        return net_returns

def analyze_ab_test(control, treatment, segments):
    """
    分层分析A/B测试结果
    """
    results = []

    # 总体效果
    overall = {
        'segment': 'overall',
        'control_mean': control['returns'].mean(),
        'treatment_mean': treatment['returns'].mean(),
        'lift': (treatment['returns'].mean() - control['returns'].mean())
                / control['returns'].mean(),
        'p_value': ttest_ind(treatment['returns'], control['returns'])[1]
    }
    results.append(overall)

    # 分层效果
    for segment in segments:
        ctrl_seg = control[control['segment'] == segment]
        treat_seg = treatment[treatment['segment'] == segment]

        seg_result = {
            'segment': segment,
            'control_mean': ctrl_seg['returns'].mean(),
            'treatment_mean': treat_seg['returns'].mean(),
            'lift': (treat_seg['returns'].mean() - ctrl_seg['returns'].mean())
                    / ctrl_seg['returns'].mean(),
            'p_value': ttest_ind(treat_seg['returns'], ctrl_seg['returns'])[1]
        }
        results.append(seg_result)

    return pd.DataFrame(results)

背景：某消费金融公司的信用评分模型上线一年后，发现通过率稳定但违约率上升。

诊断过程
1. 检查模型性能指标
   KS值：0.35 → 0.28 (下降)
   AUC：0.78 → 0.72 (下降)

2. 检查特征分布
   PSI分析发现：
   - 收入特征 PSI = 0.32 (显著漂移)
   - 年龄特征 PSI = 0.08 (稳定)
   - 负债比特征 PSI = 0.28 (显著漂移)

3. 根因分析
   - 疫情后客群结构变化
   - 新渠道带来不同客群
   - 宏观经济环境变化
解决方案
调整阈值
中期措施: 增加新特征
长期措施: 建立自动化监控
人工审核高风险段
中期措施: 重新训练模型
长期措施: 建立模型更新机制
限制高风险渠道
中期措施: A/B测试新模型
长期措施: 建立模型治理流程

Q1: 为什么回测收益不能代表未来收益？

A: 回测与实际的差距来源：

过拟合
机制: 策略针对历史优化
缓解方法: 样本外测试、交叉验证
未来信息
机制: 使用了决策时不可用的数据
缓解方法: 严格检查数据时间戳
市场冲击
机制: 实际交易会影响价格
缓解方法: 模拟市场冲击成本
市场变化
机制: 历史规律不再成立
缓解方法: 多周期验证、理论支撑
生存偏差
机制: 只看到存活的标的
缓解方法: 使用完整历史数据

Q2: A/B测试中如何处理"新奇效应"？

A: 新奇效应指用户对新功能的短期兴趣，可能导致高估长期效果。

解决方案：

1.延长实验周期：观察效果是否随时间衰减
2.分析时间趋势：看指标是否有下降趋势
3.老用户 vs 新用户：分开分析
4.使用长期指标：如6个月留存而非7天留存

Q3: 模型漂移检测到后应该怎么办？

A: 分级响应机制：

轻微
判断标准: PSI<0.1，性能下降<5%
响应措施: 加强监控，记录观察
中度
判断标准: 0.1
响应措施: 启动模型调优，准备更新
严重
判断标准: PSI>0.25，性能下降>15%
响应措施: 降级服务，人工介入，紧急更新

Q4: 如何设计一个稳健的回测系统？

A: 关键设计原则：

python
class RobustBacktestSystem:
    """稳健回测系统的设计要点"""

    def __init__(self):
        self.checks = [
            'no_lookahead_bias',      # 无未来信息
            'realistic_costs',         # 真实成本
            'proper_data_split',       # 正确数据分割
            'statistical_tests',       # 统计检验
            'multiple_periods',        # 多周期
            'parameter_sensitivity',   # 参数敏感性
        ]

    def validate_strategy(self, strategy):
        """策略验证流程"""
        results = {}

        # 1. 检查是否使用未来信息
        results['lookahead'] = self.check_lookahead(strategy)

        # 2. 样本外测试
        results['oos_performance'] = self.out_of_sample_test(strategy)

        # 3. 参数敏感性
        results['sensitivity'] = self.parameter_sensitivity(strategy)

        # 4. 统计显著性
        results['significance'] = self.statistical_significance(strategy)

        # 5. 多周期表现
        results['period_consistency'] = self.multi_period_test(strategy)

        return results

Q5: 金融场景的A/B测试有哪些特殊注意事项？

A: 金融A/B测试的特殊性：

合规审查
原因: 金融服务受监管
建议: 法务和合规提前介入
用户告知
原因: 涉及资金安全
建议: 必要时知情同意
风险控制
原因: 可能造成损失
建议: 设置熔断机制
长周期
原因: 金融决策影响长期
建议: 延长观察期
外部因素
原因: 市场环境影响大
建议: 控制或调整
