向量化（Embedding）

Embedding的本质

将文本映射到高维向量空间，语义相似的文本在向量空间中距离较近。

\[\text{Embedding}: \text{Text} \rightarrow \mathbb{R}^d\]

def retrieve_with_rerank(query, top_k=5, rerank_top_n=20):
    # 1. 粗筛：快速召回更多候选
    candidates = vector_db.search(query, top_k=rerank_top_n)

    # 2. 精排：用更强的模型重排序
    reranked = reranker.rerank(query, candidates)

    # 3. 返回Top-K
    return reranked[:top_k]

检索评估指标

公式: \(\frac{\text{相关文档 in Top-K}}{\text{总相关文档}}\)
含义: 召回率
Precision@K
公式: \(\frac{\text{相关文档 in Top-K}}{K}\)
含义: 精确率
MRR
公式: \(\frac{1}{\text{第一个相关文档的排名}}\)
含义: 首个相关结果的排名
NDCG
公式: 考虑排名位置的相关性加权
含义: 整体排序质量

模型微调（Fine-tuning）

微调的目的
•适应特定领域术语和风格
•提升特定任务性能
•减少推理时的提示长度
微调方法对比
方法训练参数显存需求效果金融场景
全参数微调100%极高最好大机构
LoRA0.1-1%低很好推荐
QLoRA0.1-1%更低很好资源受限
Prompt Tuning<0.1%最低一般快速实验
Prefix Tuning<0.1%低一般多任务
LoRA原理
\[W' = W + BA\]

其中：

•\(W\): 原始权重矩阵 \((d \times d)\)
•\(B\): 低秩矩阵 \((d \times r)\)
•\(A\): 低秩矩阵 \((r \times d)\)
•\(r\): 秩，通常 \(r << d\)

    sections = re.split(pattern, text)

RAG系统完整实现

python
class FinancialRAG:
    def __init__(self, embedding_model, vector_db, llm):
        self.embedder = embedding_model
        self.db = vector_db
        self.llm = llm

    def ingest(self, documents):
        """文档入库"""
        for doc in documents:
            # 1. 分块
            chunks = self.chunk(doc)
            # 2. 向量化
            embeddings = self.embedder.embed(chunks)
            # 3. 存储
            self.db.add(chunks, embeddings, metadata=doc.metadata)

    def query(self, question, top_k=5):
        """查询回答"""
        # 1. 检索
        contexts = self.retrieve(question, top_k)

        # 2. 构建提示
        prompt = self.build_prompt(question, contexts)

        # 3. 生成
        answer = self.llm.generate(prompt)

        # 4. 添加引用
        return self.add_citations(answer, contexts)

    def retrieve(self, query, top_k):
        """混合检索"""
        # 向量检索
        vector_results = self.db.vector_search(query, top_k * 2)
        # 关键词检索
        keyword_results = self.db.keyword_search(query, top_k * 2)
        # 融合重排序
        merged = self.merge_and_rerank(vector_results, keyword_results, query)
        return merged[:top_k]

    def build_prompt(self, question, contexts):
        context_text = "\n\n".join([c.content for c in contexts])
        return f"""基于以下金融资料回答问题。

参考资料：
{context_text}

问题：{question}

要求：
1. 仅基于参考资料回答
2. 如果资料中没有相关信息，明确说明
3. 引用具体来源
"""

def chunk_research_report(report):
    chunks = []
    # 1. 保留元数据
    metadata = {
        "title": report.title,
        "date": report.date,
        "analyst": report.analyst,
        "industry": report.industry
    }

    # 2. 按章节分块
    for section in report.sections:
        section_chunks = text_splitter.split(section.content)
        for chunk in section_chunks:
            chunk.metadata = {
                **metadata,
                "section": section.title
            }
            chunks.append(chunk)

    return chunks

经验法则
chunk_size ≈ 0.5 × embedding_model_max_length
overlap ≈ 0.1 × chunk_size

Q4: 如何提升RAG的检索准确率？

A: 分层优化策略：

1. 数据层

•提升分块质量，保留完整语义
•添加丰富的元数据
•数据清洗和去重

2. 索引层

•使用更好的Embedding模型
•混合检索（向量+关键词）
•为不同类型数据建立专属索引

3. 检索层

•增加召回数量 + 重排序
•查询改写和扩展
•使用HyDE等高级策略

4. 生成层

•优化提示词模板
•强调基于上下文回答
•添加事实核查

Q7: RAG系统的局限性是什么？

A:

检索瓶颈
描述: 检索不到就无法回答
应对策略: 混合检索、多路召回
长推理链
描述: 需要多步推理时困难
应对策略: Agent架构、思维链
矛盾处理
描述: 多个来源信息矛盾
应对策略: 冲突检测、来源权重
实时性
描述: 知识库更新有延迟
应对策略: 流式更新、缓存策略
成本
描述: Embedding和检索有成本
应对策略: 缓存、异步、批量

Q8: 金融场景微调有哪些风险？

A:

灾难性遗忘
描述: 丢失通用能力
缓解措施: 混合数据训练
过拟合
描述: 训练集表现好、泛化差
缓解措施: 早停、验证集
数据泄露
描述: 敏感信息泄露
缓解措施: 数据脱敏、访问控制
偏见放大
描述: 放大训练数据偏见
缓解措施: 偏见检测、平衡数据
监管合规
描述: 模型行为不可预测
缓解措施: 测试、监控、审计
